## Set（集合）

### 什么是 Set？

Set（集合）是 Redis 中一种**无序且元素唯一**的数据结构，类似于数学中的“集合”概念。

你可以把它想象成一个**抽奖箱**：
- 每个球上写着一个名字
- 同一个人的名字只能放一次（去重）
- 抽奖时随机抽出，不按顺序

Set 非常适合用于“去重”、“标签管理”、“共同好友”等场景。

---

### 通俗易懂的解释

想象你有一堆贴纸，每张贴纸上写了一个兴趣标签，比如：

```
"跑步"、"读书"、"电影"、"旅行"、"跑步"
```

如果你把这些贴纸放进一个 **Set 容器**：
- 重复的“跑步”只会保留一个
- 贴纸是乱序的，没有先后
- 你想查“有没有‘游泳’这个标签？”，一秒就能知道

这就是 Redis Set 的核心思想：**无序 + 唯一 + 快速查询**

---

### 核心特性

| 特性 | 说明 |
|------|------|
| **无序性** | 元素没有固定顺序（Redis 3.2+ 使用 intset 或 hashtable 优化） |
| **唯一性** | 同一个元素只能存在一次，自动去重 |
| **高效操作** | 添加、删除、查询时间复杂度接近 O(1) |
| **支持集合运算** | 并集、交集、差集等数学集合操作 |
| **不支持重复元素** | 不像 List，Set 中不能有重复 member |

底层使用 **哈希表（Hash Table）** 或 **整数集合（intset）** 实现，兼顾性能与内存。

---

### 常用命令

#### 1. `SADD key member [member ...]`
- **作用**：向集合添加一个或多个元素。
- **示例**：
  ```bash
  SADD user:1001:tags "跑步" "读书" "电影"
  ```
  返回值是**成功添加的新元素个数**（重复的不计入）。

---

#### 2. `SMEMBERS key`
- **作用**：获取集合中所有元素（无序）。
- **示例**：
  ```bash
  SMEMBERS user:1001:tags
  ```
  返回：`["电影", "读书", "跑步"]`（顺序可能不同）

> ⚠️ 数据量大时慎用，会阻塞 Redis！

---

#### 3. `SISMEMBER key member`
- **作用**：判断某个元素是否在集合中。
- **示例**：
  ```bash
  SISMEMBER user:1001:tags "跑步"
  ```
  返回 `1`（存在）或 `0`（不存在） ✅ 推荐用于“是否存在”判断

---

#### 4. `SCARD key`
- **作用**：返回集合的元素个数（基数）。
- **示例**：
  ```bash
  SCARD user:1001:tags  # 返回 3
  ```

---

#### 5. `SREM key member [member ...]`
- **作用**：从集合中删除一个或多个元素。
- **示例**：
  ```bash
  SREM user:1001:tags "电影"
  ```
  返回成功删除的个数。

---

#### 6. `SPOP key [count]`
- **作用**：随机弹出并移除一个或多个元素（可用于抽奖）。
- **示例**：
  ```bash
  SPOP user:1001:tags  # 随机返回并删除一个标签
  ```

---

#### 7. `SRANDMEMBER key [count]`
- **作用**：随机获取元素，但**不删除**。
- **示例**：
  ```bash
  SRANDMEMBER user:1001:tags 2  # 随机返回 2 个标签
  ```

---

#### 8. `集合运算命令`

- `SINTER key [key ...]`：求多个集合的**交集**  
  ```bash
  SINTER user:1001:tags user:1002:tags  # 找共同兴趣
  ```

- `SUNION key [key ...]`：求**并集**  
  ```bash
  SUNION user:1001:tags user:1002:tags  # 所有兴趣合集
  ```

- `SDIFF key [key ...]`：求**差集**（第一个集合有，其他没有）  
  ```bash
  SDIFF user:1001:tags user:1002:tags  # 1001 有但 1002 没有的标签
  ```

### 🌰 实际应用场景举例：社交系统标签与好友推荐

#### 场景 1：用户兴趣标签管理
```bash
# 用户添加兴趣
SADD user:1001:interests "科技" "编程" "音乐"

# 查看用户兴趣
SMEMBERS user:1001:interests

# 判断是否感兴趣
SISMEMBER user:1001:interests "编程"  # 返回 1
```

#### 场景 2：找共同好友
```bash
SADD user:1001:friends 2001 2002 2003
SADD user:1002:friends 2002 2003 2004

# 找共同好友
SINTER user:1001:friends user:1002:friends
# 返回：2002, 2003
```

#### 场景 3：抽奖系统
```bash
# 所有参与抽奖的用户 ID
SADD lottery:users 1001 1002 1003 1004 1005

# 随机抽 1 人中奖（并移除）
SPOP lottery:users
```

### ✅ Set 的经典使用场景

| 场景 | 说明 |
|------|------|
| **去重** | 记录用户搜索关键词、页面访问 IP，自动去重 |
| **标签系统** | 用户标签、商品标签、文章分类 |
| **共同好友/兴趣** | 社交网络中求交集 |
| **抽奖/随机抽取** | `SPOP` 实现随机中奖 |
| **权限控制** | 用户角色集合，判断是否有某权限 |
| **临时集合计算** | 快速做交并差，替代数据库复杂查询 |

---

### 🔔 注意事项

- `SMEMBERS` 返回所有元素，**大数据量会阻塞 Redis**，建议用 `SSCAN` 迭代
- Set 不保证顺序，如需排序请用 **Zset**
- 元素是字符串，最大 512MB
- 内存占用比 List 略高（因需维护哈希表去重）

---

### ✅ 总结：Set 适合什么场景？

| 特点 | 是否适合 Set |
|------|----------------|
| 需要去重 | ✅ 核心优势 |
| 元素唯一 | ✅ 自动保证 |
| 快速判断是否存在 | ✅ `SISMEMBER` O(1) |
| 做集合运算（交并差） | ✅ 原生支持 |
| 数据无序 | ✅ 无序是常态 |
| 随机抽取 | ✅ `SPOP` / `SRANDMEMBER` |

---

**只要你的需求是“有哪些不重复的项？”、“有没有这个东西？”、“谁和我有共同爱好？”，Redis Set 就是那个简单、高效、可靠的答案！**
