## 享元模式

享元模式（Flyweight Pattern）是一种结构型设计模式，旨在最小化内存使用量或减少对象创建的数量，通过共享尽可能多的数据来实现。它特别适用于处理大量细粒度的对象，这些对象之间存在大量的重复数据，而这些重复数据可以被多个对象共享。

### 核心思想

享元模式的核心在于区分对象的内部状态和外部状态：

- **内部状态**：可以在多个对象实例间共享的状态，通常是不变的。
- **外部状态**：随环境变化而变化的状态，不能被共享，通常由客户端在需要时传递给享元对象。

通过这种方式，享元模式能够显著减少应用程序中对象的数量，从而降低内存消耗，并可能提高性能。

### 享元模式的主要组件

1. **Flyweight（抽象享元角色）**：定义了一个接口，所有具体享元类都必须实现此接口。这个角色是可选的，但有助于实现更灵活的设计。

2. **ConcreteFlyweight（具体享元角色）**：实现了Flyweight接口，包含了可以共享的部分。需要注意的是，具体享元角色不能修改其内部状态。

3. **UnsharedConcreteFlyweight（不可共享的具体享元角色）**：并非所有的享元对象都可以被共享，某些情况下，某些对象可能包含无法共享的数据，这类对象就是不可共享的具体享元角色。

4. **FlyweightFactory（享元工厂角色）**：负责创建并管理享元对象，确保相同的享元对象只创建一次，并提供访问已创建对象的方法。它通常会维护一个享元池，用来存储已经创建的享元对象。

### 使用场景

- 当系统中有大量相似的对象时，使用享元模式可以有效减少内存占用。
- 需要缓存大量对象以提高性能的应用程序。
- 字体渲染、文档编辑器中的字符表示等场景非常适合使用享元模式。

### 示例代码

以下是一个简单的示例，展示了如何使用享元模式来优化字符串的存储：

```java
import java.util.HashMap;

// Flyweight interface
interface Character {
    void display(char externalState);
}

// ConcreteFlyweight
class ConcreteCharacter implements Character {
    private char internalState;

    public ConcreteCharacter(char internalState) {
        this.internalState = internalState;
    }

    @Override
    public void display(char externalState) {
        System.out.println("Character: " + internalState + ", External State: " + externalState);
    }
}

// Flyweight factory
class CharacterFactory {
    private HashMap<Character, ConcreteCharacter> characters = new HashMap<>();

    public Character getCharacter(char key) {
        if (!characters.containsKey(key)) {
            characters.put(key, new ConcreteCharacter(key));
        }
        return characters.get(key);
    }
}

public class FlyweightDemo {
    public static void main(String[] args) {
        CharacterFactory factory = new CharacterFactory();

        // 假设我们有重复的字符需要显示
        String document = "aabbcc";

        for (char ch : document.toCharArray()) {
            Character character = factory.getCharacter(ch);
            character.display(' '); // 这里可以传递不同的外部状态
        }
    }
}
```

在这个例子中，`CharacterFactory` 负责创建和管理 `ConcreteCharacter` 实例，避免了相同字符的重复创建，从而节省了内存空间。每个 `ConcreteCharacter` 对象只保存字符本身作为内部状态，而展示位置等信息则作为外部状态传递给 `display()` 方法。
