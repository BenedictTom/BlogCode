## 创建线程的三种方式

#### Thread

Thread 创建线程方式：创建线程类，匿名内部类方式

* **start() 方法底层其实是给 CPU 注册当前线程，并且触发 run() 方法执行**
* 线程的启动必须调用 start() 方法，如果线程直接调用 run() 方法，相当于变成了普通类的执行，此时主线程将只有执行该线程
* 建议线程先创建子线程，主线程的任务放在之后，否则主线程（main）永远是先执行完

Thread 构造器：

* `public Thread()`
* `public Thread(String name)`

```java
public class ThreadDemo {
    public static void main(String[] args) {
        Thread t = new MyThread();
        t.start();
       	for(int i = 0 ; i < 100 ; i++ ){
            System.out.println("main线程" + i)
        }
        // main线程输出放在上面 就变成有先后顺序了，因为是 main 线程驱动的子线程运行
    }
}
class MyThread extends Thread {
    @Override
    public void run() {
        for(int i = 0 ; i < 100 ; i++ ) {
            System.out.println("子线程输出："+i)
        }
    }
}
```

继承 Thread 类的优缺点：

* 优点：编码简单
*  缺点：线程类已经继承了 Thread 类无法继承其他类了，功能不能通过继承拓展（单继承的局限性）

#### Runnable

Runnable 创建线程方式：创建线程类，匿名内部类方式

Thread 的构造器：

* `public Thread(Runnable target)`
* `public Thread(Runnable target, String name)`

```java
public class ThreadDemo {
    public static void main(String[] args) {
        Runnable target = new MyRunnable();
        Thread t1 = new Thread(target,"1号线程");
		t1.start();
        Thread t2 = new Thread(target);//Thread-0
    }
}

public class MyRunnable implements Runnable{
    @Override
    public void run() {
        for(int i = 0 ; i < 10 ; i++ ){
            System.out.println(Thread.currentThread().getName() + "->" + i);
        }
    }
}
```

**Thread 类本身也是实现了 Runnable 接口**，Thread 类中持有 Runnable 的属性，执行线程 run 方法底层是调用 Runnable#run：

```java
public class Thread implements Runnable {
    private Runnable target;
    
    public void run() {
        if (target != null) {
          	// 底层调用的是 Runnable 的 run 方法
            target.run();
        }
    }
}
```

Runnable 方式的优缺点：

* 缺点：代码复杂一点。

* 优点：

  1. 线程任务类只是实现了 Runnable 接口，可以继续继承其他类，避免了单继承的局限性

  2. 同一个线程任务对象可以被包装成多个线程对象

  3. 适合多个多个线程去共享同一个资源

  4. 实现解耦操作，线程任务代码可以被多个线程共享，线程任务代码和线程独立

  5. 线程池可以放入实现 Runnable 或 Callable 线程任务对象

#### Callable

实现 Callable 接口：

1. 定义一个线程任务类实现 Callable 接口，申明线程执行的结果类型
2. 重写线程任务类的 call 方法，这个方法可以直接返回执行的结果
3. 创建一个 Callable 的线程任务对象
4. 把 Callable 的线程任务对象**包装成一个未来任务对象**
5. 把未来任务对象包装成线程对象
6. 调用线程的 start() 方法启动线程

`public FutureTask(Callable<V> callable)`：未来任务对象，在线程执行完后得到线程的执行结果

* FutureTask 就是 Runnable 对象，因为 **Thread 类只能执行 Runnable 实例的任务对象**，所以把 Callable 包装成未来任务对象
* 线程池部分详解了 FutureTask 的源码

`public V get()`：同步等待 task 执行完毕的结果，如果在线程中获取另一个线程执行结果，会阻塞等待，用于线程同步

* get() 线程会阻塞等待任务执行完成
* run() 执行完后会把结果设置到 FutureTask  的一个成员变量，get() 线程可以获取到该变量的值

优缺点：

* 优点：同 Runnable，并且能得到线程执行的结果
* 缺点：编码复杂

```java
public class ThreadDemo {
    public static void main(String[] args) {
        Callable call = new MyCallable();
        FutureTask<String> task = new FutureTask<>(call);
        Thread t = new Thread(task);
        t.start();
        try {
            String s = task.get(); // 获取call方法返回的结果（正常/异常结果）
            System.out.println(s);
        }  catch (Exception e) {
            e.printStackTrace();
        }
    }

public class MyCallable implements Callable<String> {
    @Override//重写线程任务类方法
    public String call() throws Exception {
        return Thread.currentThread().getName() + "->" + "Hello World";
    }
}
```

## FutureTask 与 Future

`FutureTask` 和 `Future` 是 Java 并发包（`java.util.concurrent`）中的两个重要组件，用于处理异步任务的执行结果。它们为获取异步操作的结果提供了一种机制，允许你检查任务是否完成、等待其完成并检索结果。

### Future

`Future` 是一个接口，它表示异步计算的结果。提供了检查计算是否完成、等待计算完成和获取计算结果的方法。无法直接实例化 `Future`，而是通过如 `ExecutorService.submit()` 方法来获得它的实现类对象。`Future` 的主要方法包括：

- **boolean isDone()**：如果任务完成，则返回 `true`。
- **V get() throws InterruptedException, ExecutionException**：等待计算完成，并获取结果。
- **V get(long timeout, TimeUnit unit) throws InterruptedException, ExecutionException, TimeoutException**：在指定时间内等待计算完成，并获取结果。
- **boolean cancel(boolean mayInterruptIfRunning)**：尝试取消执行的任务。
- **boolean isCancelled()**：若此任务在正常完成前被取消，则返回 `true`。

### FutureTask

`FutureTask` 是 `Future` 接口的一个具体实现类，同时实现了 `Runnable` 接口。因此，它可以作为单独的线程执行单元提交给 `Thread` 来运行，也可以提交给 `Executor` 执行。它封装了一个 `Callable` 或 `Runnable` 对象，并且可以通过 `get` 方法获取执行结果或者异常。

使用 `FutureTask` 时，通常需要按照以下步骤进行：

1. 创建一个 `Callable` 实例或 `Runnable` 实例。
2. 使用这个实例创建 `FutureTask`。
3. 将 `FutureTask` 提交给 `Executor` 或者包装成 `Thread` 对象启动。

**示例代码：**

```java
import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.FutureTask;

public class Main {
    public static void main(String[] args) {
        Callable<Integer> callable = () -> {
            System.out.println("正在执行计算...");
            Thread.sleep(5000);
            return 123;
        };

        FutureTask<Integer> futureTask = new FutureTask<>(callable);
        Thread thread = new Thread(futureTask);
        thread.start();

        try {
            System.out.println("计算结果：" + futureTask.get());
        } catch (InterruptedException | ExecutionException e) {
            e.printStackTrace();
        }
    }
}
```

在这个例子中，我们定义了一个 `Callable`，它模拟了一个耗时操作，并最终返回整数结果。然后创建了一个 `FutureTask`，将其作为参数传递给一个新的 `Thread` 对象，并启动该线程。最后通过调用 `futureTask.get()` 方法等待任务完成并获取结果。

总之，`FutureTask` 提供了对 `Callable` 和 `Runnable` 的封装，并且实现了 `Future` 接口的功能，使得你可以方便地管理异步任务的执行过程和结果。




## Executor
### Executor 概述

`Executor` 是 Java 中用于管理和调度线程的一个框架，它位于 `java.util.concurrent` 包中。`Executor` 提供了一种标准的方法将任务的提交与任务的执行分离开来。通过使用 `Executor`，你可以更灵活地控制任务如何被并发执行，而不需要直接管理线程的创建和销毁。

`Executor` 接口非常简单，只有一个方法：

```java
void execute(Runnable command);
```

尽管 `Executor` 可以用来执行任何实现 `Runnable` 接口的任务，但通常它会被扩展为更复杂的接口或类，如 `ExecutorService` 和 `ScheduledExecutorService`，它们提供了更多的功能，包括管理线程池、任务取消、任务完成检测等。

### 线程池

是的，`Executor` 通常与线程池关联。线程池是一种设计模式，它维护一个线程的集合（称为“池”），这些线程可以重复使用来执行多个任务。这减少了由于频繁创建和销毁线程带来的开销，并有助于提高性能。Java 提供了 `ThreadPoolExecutor` 类以及一些方便的方法如 `Executors.newFixedThreadPool`, `Executors.newCachedThreadPool` 来创建不同类型的线程池。

### Future 获得返回结果

当你通过 `ExecutorService` 的 `submit()` 方法提交一个任务时，无论是实现了 `Runnable` 还是 `Callable` 接口的任务，都会返回一个 `Future` 对象。这个对象代表了异步计算的结果。

- **对于实现了 `Runnable` 的任务**：因为 `Runnable` 的 `run()` 方法不返回结果，所以对应的 `Future.get()` 方法总是返回 `null`。
- **对于实现了 `Callable` 的任务**：`Callable.call()` 方法可以返回一个结果，这个结果可以通过 `Future.get()` 方法获得。此外，如果任务抛出了异常，调用 `get()` 方法时会抛出 `ExecutionException`。

例如，下面是如何通过 `Future` 获得一个实现了 `Callable` 接口的任务的结果：

```java
import java.util.concurrent.*;

public class Main {
    public static void main(String[] args) throws InterruptedException, ExecutionException {
        ExecutorService executor = Executors.newSingleThreadExecutor();
        
        Callable<Integer> task = () -> {
            TimeUnit.SECONDS.sleep(1);
            return 123;
        };
        
        Future<Integer> future = executor.submit(task);
        
        System.out.println("任务是否完成: " + future.isDone());
        
        Integer result = future.get(); // 阻塞直到任务完成
        
        System.out.println("任务是否完成: " + future.isDone());
        System.out.println("结果: " + result);
        
        executor.shutdown();
    }
}
```

在这个例子中，我们创建了一个 `Callable` 任务并将其提交给 `ExecutorService`。然后，我们调用 `future.get()` 方法来等待任务完成并获取其结果。注意，`get()` 方法是阻塞的，直到任务完成为止。如果任务完成了，则 `isDone()` 方法返回 `true`。



## ExecutorService的概念
`ExecutorService` 是 Java 中用于管理线程执行任务的核心接口之一，是 `java.util.concurrent` 包中的重要组成部分。它是对 `Executor` 接口的扩展，提供了更丰富的功能，比如：

- 提交异步任务（支持返回值）
- 管理线程池
- 控制任务调度
- 关闭和优雅停止线程池

## ⚙️ 三、常见使用方式

### ✅ 1. 创建线程池（通过 Executors 工厂类）

Java 提供了 `Executors` 工具类来创建常见的线程池类型：

#### 常见线程池类型：

| 类型 | 描述 |
|------|------|
| `newFixedThreadPool(int nThreads)` | 固定大小的线程池 |
| `newCachedThreadPool()` | 缓存线程池，根据需要创建新线程，空闲线程会超时回收 |
| `newSingleThreadExecutor()` | 单线程的线程池，保证顺序执行任务 |
| `newScheduledThreadPool(int corePoolSize)` | 支持定时和周期性任务的线程池 |

**示例：**

```java
ExecutorService executor = Executors.newFixedThreadPool(4);
```

---

### ✅ 2. 提交任务（Runnable / Callable）

#### 使用 `execute()` 提交 Runnable（无返回值）：

```java
executor.execute(() -> {
    System.out.println("任务执行");
});
```

#### 使用 `submit()` 提交任务并获取 Future 结果：

```java
Future<String> future = executor.submit(() -> "Hello from Callable");

String result = future.get(); // 阻塞直到任务完成
System.out.println(result);
```

---

### ✅ 3. 关闭线程池

在程序结束前，应该主动关闭线程池，否则主线程可能不会退出。

```java
executor.shutdown(); // 不再接受新任务，等待已提交任务完成

// 或者强制关闭：
List<Runnable> notExecutedTasks = executor.shutdownNow(); // 尝试中断正在运行的任务
```

## 🔄 四、线程池内部机制简要说明

线程池本质上维护了一个 **任务队列** 和一组 **工作线程**。

- 当你调用 `submit()` 或 `execute()` 提交任务时：
  - 如果当前线程数 < 核心线程数，直接创建新线程处理任务。
  - 如果线程数 >= 核心线程数，则将任务放入队列中等待。
  - 如果队列满且线程数 < 最大线程数，继续创建线程。
  - 如果队列满且线程数 >= 最大线程数，拒绝任务（默认抛异常）。

## 🧠 五、为什么推荐使用 ExecutorService？

| 对比项 | 直接 new Thread() | 使用 ExecutorService |
|--------|------------------|---------------------|
| 资源开销 | 每次新建线程，资源浪费 | 复用线程，性能更好 |
| 并发控制 | 难以管理并发数量 | 可控线程池大小 |
| 异常处理 | 不易统一处理 | 可通过 Future 获取异常 |
| 生命周期管理 | 自己管理线程 | 提供 shutdown 等方法 |
| 可扩展性 | 很差 | 支持定时、异步、组合等高级操作 |

```java
import java.util.concurrent.*;

public class ExecutorServiceDemo {

        System.out.println("任务结果: " + result);

        executor.shutdown();
    }
}
```

## 📌 总结

`ExecutorService` 是现代 Java 并发编程中非常重要的一个组件，它不仅简化了线程管理，还提高了程序的性能和可维护性。相比直接使用 `new Thread().start()`，使用线程池和 `ExecutorService` 是一种更加专业和推荐的做法。



## Java 中启动线程或执行任务的所有常见方式。


你提到的几种方式确实都可以启动线程或执行任务，但它们在本质和使用场景上略有不同。我们可以系统地梳理一下 **Java 中启动线程或执行任务的所有常见方式**。

---

## ✅ 一、创建线程的标准方法（底层直接操作线程）

### 方法1：继承 `Thread` 类并调用 `start()`

```java
Thread t = new Thread() {
    public void run() {
        System.out.println("线程运行");
    }
};
t.start(); // 启动新线程
```

- **特点**：直接创建线程对象，并通过 `start()` 启动线程。
- **不推荐**：耦合了任务与线程本身，不利于复用和管理。

---

### 方法2：实现 `Runnable` 接口 + `Thread.start()`

```java
Runnable task = () -> System.out.println("线程运行");
Thread t = new Thread(task);
t.start();
```

- **推荐方式**：将任务与线程分离，更灵活。
- **适合简单并发任务**。

---

## ✅ 二、使用并发工具类/框架的方式（推荐现代做法）

### 方法3：使用 `ExecutorService` 提交任务

```java
ExecutorService executor = Executors.newSingleThreadExecutor();
executor.execute(() -> System.out.println("任务执行"));
executor.shutdown();
```

- **说明**：`execute()` 提交的是 `Runnable`，不会返回结果。
- **优点**：线程池管理线程资源，提升性能、简化并发控制。

---

### 方法4：使用 `ExecutorService.submit()` + `FutureTask` 或 `Callable`

```java
ExecutorService executor = Executors.newSingleThreadExecutor();
Future<?> future = executor.submit(() -> System.out.println("任务执行"));
executor.shutdown();
```

- 可以提交 `Runnable` 或 `Callable`，并获取 `Future` 对象。
- `Callable` 支持返回值和异常处理。

---

### 方法5：直接调用 `FutureTask.run()`（不推荐）

```java
Callable<String> callable = () -> "完成";
FutureTask<String> task = new FutureTask<>(callable);
new Thread(task).start(); // 正确做法
// task.run(); // 不推荐！这是同步调用，不会开启新线程
```

- ❗注意：直接调用 `task.run()` 是在当前线程同步执行，不会新开线程。
- ✅正确做法是包装进 `Thread` 并调用 `start()`。

---

## ✅ 三、其他现代异步编程方式（Java 8+）

### 方法6：使用 `CompletableFuture` 异步执行任务

```java
CompletableFuture<Void> future = CompletableFuture.runAsync(() -> {
    System.out.println("异步执行");
});
future.join(); // 等待完成
```

- **优点**：支持链式调用、组合多个任务、回调等。
- **推荐用于复杂异步流程**。

---

### 方法7：使用 `ForkJoinPool`（适用于并行流等）

```java
ForkJoinPool pool = new ForkJoinPool();
pool.execute(() -> System.out.println("任务执行"));
pool.shutdown();
```

- 适用于分治算法、并行流等高级并发场景。

---

## 📌 总结：启动线程或执行任务的方式汇总

| 方式 | 是否真正启动线程 | 是否推荐 | 使用场景 |
|------|------------------|----------|-----------|
| `Thread.start()` | ✅ | ⚠️ 一般 | 基础线程操作 |
| `Runnable + Thread.start()` | ✅ | ✅ 推荐 | 常规并发任务 |
| `Executor.execute(Runnable)` | ✅（由线程池） | ✅ 推荐 | 简单任务调度 |
| `Executor.submit(Callable)` | ✅ | ✅ 推荐 | 需要返回值的任务 |
| `FutureTask.run()` | ❌（同步执行） | ❌ | ❌ 不推荐 |
| `CompletableFuture.runAsync()` | ✅ | ✅ 推荐 | 异步流程编排 |
| `ForkJoinPool.execute()` | ✅ | ✅ 推荐 | 并行计算、分治任务 |

---

## 🔍 为什么不能用 `futureTask.run()` 来“启动线程”？

因为：

- `run()` 方法只是一个普通方法调用，它会在**当前线程中同步执行**。
- 它不会创建新的线程。
- 要想异步执行，必须把 `FutureTask` 包装成 `Thread` 并调用 `start()`。

## 线程方法

#### API

Thread 类 API：

| 方法                                        | 说明                                                         |
| ------------------------------------------- | ------------------------------------------------------------ |
| public void start()                         | 启动一个新线程，Java虚拟机调用此线程的 run 方法              |
| public void run()                           | 线程启动后调用该方法                                         |
| public void setName(String name)            | 给当前线程取名字                                             |
| public void getName()                       | 获取当前线程的名字<br />线程存在默认名称：子线程是 Thread-索引，主线程是 main |
| public static Thread currentThread()        | 获取当前线程对象，代码在哪个线程中执行                       |
| public static void sleep(long time)         | 让当前线程休眠多少毫秒再继续执行<br />**Thread.sleep(0)** : 让操作系统立刻重新进行一次 CPU 竞争 |
| public static native void yield()           | 提示线程调度器让出当前线程对 CPU 的使用                      |
| public final int getPriority()              | 返回此线程的优先级                                           |
| public final void setPriority(int priority) | 更改此线程的优先级，常用 1 5 10                              |
| public void interrupt()                     | 中断这个线程，异常处理机制                                   |
| public static boolean interrupted()         | 判断当前线程是否被打断，清除打断标记                         |
| public boolean isInterrupted()              | 判断当前线程是否被打断，不清除打断标记                       |

sleep：

#### join

public final void join()：等待这个线程结束

原理：调用者轮询检查线程 alive 状态，t1.join() 等价于：

```java
public final synchronized void join(long millis) throws InterruptedException {
    // 调用者线程进入 thread 的 waitSet 等待, 直到当前线程运行结束
    while (isAlive()) {
        wait(0);
    }
}
```

* join 方法是被 synchronized 修饰的，本质上是一个对象锁，其内部的 wait 方法调用也是释放锁的，但是**释放的是当前的线程对象锁，而不是外面的锁**

* 当调用某个线程（t1）的 join 方法后，该线程（t1）抢占到 CPU 资源，就不再释放，直到线程执行完毕

线程同步：

* join 实现线程同步，因为会阻塞等待另一个线程的结束，才能继续向下运行
  * 需要外部共享变量，不符合面向对象封装的思想
  * 必须等待线程结束，不能配合线程池使用
* Future 实现（同步）：get() 方法阻塞等待执行结果
  * main 线程接收结果
  * get 方法是让调用线程同步等待

##### 终止模式

终止模式之两阶段终止模式：Two Phase Termination

目标：在一个线程 T1 中如何优雅终止线程 T2？优雅指的是给 T2 一个后置处理器

错误思想：

* 使用线程对象的 stop() 方法停止线程：stop 方法会真正杀死线程，如果这时线程锁住了共享资源，当它被杀死后就再也没有机会释放锁，其它线程将永远无法获取锁
* 使用 System.exit(int) 方法停止线程：目的仅是停止一个线程，但这种做法会让整个程序都停止

class TwoPhaseTermination {
    private Thread monitor;
    // 启动监控线程
    public void start() {
        monitor = new Thread(new Runnable() {
            @Override
            public void run() {
                while (true) {
                    Thread thread = Thread.currentThread();
                    if (thread.isInterrupted()) {
                        System.out.println("后置处理");
                        break;
                    }
                    try {
                        Thread.sleep(1000);					// 睡眠
                        System.out.println("执行监控记录");	// 在此被打断不会异常
                    } catch (InterruptedException e) {		// 在睡眠期间被打断，进入异常处理的逻辑
                        e.printStackTrace();
                        // 重新设置打断标记，打断 sleep 会清除打断状态
                        thread.interrupt();
                    }
                }
            }
        });
        monitor.start();
    }
    // 停止监控线程
    public void stop() {
        monitor.interrupt();
    }
}

## CountDownLatch
`CountDownLatch` 是 Java 并发包 `java.util.concurrent` 中的一个同步辅助类，它允许一个或多个线程等待，直到其他线程执行的一组操作完成。简单来说，`CountDownLatch` 提供了一种机制，使得某些线程可以等待直到计数器归零，这意味着所有预期的操作都已完成。

### 核心概念

- **计数器**：在创建 `CountDownLatch` 实例时需要指定一个初始计数值（即倒计数的起始值）。这个计数值代表了必须完成的工作单元数量。
- **await() 方法**：调用该方法的线程将被阻塞，直到计数器变为零。
- **countDown() 方法**：每次调用此方法都会使计数器减一。当计数器达到零时，所有因调用 `await()` 而被阻塞的线程都将被释放继续执行。

### 主要应用场景

- **并行任务启动屏障**：确保所有参与的线程准备好开始之前，主线程保持等待状态。
- **依赖关系管理**：例如，在一组并发任务完成后才能继续进行后续处理的情况。
- **资源初始化**：等待所有必要的资源加载完毕后再继续执行业务逻辑。

### 基本使用步骤

1. **初始化**：创建 `CountDownLatch` 对象，并设置初始计数值。
2. **在工作线程中调用 `countDown()`**：每完成一项工作就调用一次 `countDown()` 减少计数。
3. **在等待线程中调用 `await()`**：当需要等待所有工作完成时调用 `await()`，直到计数归零为止。

### 示例代码

以下是一个简单的例子，展示了如何使用 `CountDownLatch` 来确保几个子线程完成各自的任务之后，主线程才继续执行：

```java
import java.util.concurrent.CountDownLatch;

public class CountDownLatchDemo {
    public static void main(String[] args) throws InterruptedException {
        // 创建一个计数器，初始值为3
        CountDownLatch latch = new CountDownLatch(3);

        // 创建三个子线程，每个线程完成工作后调用 countDown()
        for (int i = 0; i < 3; i++) {
            final int taskId = i + 1;
            new Thread(() -> {
                System.out.println("子线程 " + taskId + " 开始执行...");
                try {
                    // 模拟耗时操作
                    Thread.sleep((long)(Math.random() * 1000));
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("子线程 " + taskId + " 执行完毕");
                latch.countDown(); // 子线程结束时减少计数
            }).start();
        }

        // 主线程等待所有子线程执行完毕
        System.out.println("主线程等待所有子线程完成...");
        latch.await(); // 阻塞直到计数器归零
        System.out.println("所有子线程已完成，主线程继续执行");
    }
}
```

### 关键点解释

- **`new CountDownLatch(3)`**：创建了一个计数器初值为 3 的 `CountDownLatch`，意味着我们需要等待 3 个子线程全部完成它们的任务。
- **`latch.countDown()`**：每当一个子线程完成了它的任务，就会调用 `countDown()` 方法来减少计数器的值。
- **`latch.await()`**：主线程调用 `await()` 方法，它会一直阻塞直到计数器变为零，也就是所有子线程都已经调用了 `countDown()` 方法。

### 注意事项

- **计数器不能重置**：一旦计数器达到零，所有的 `await()` 调用都不会再阻塞。如果你需要重新开始一个新的倒计数周期，你需要创建一个新的 `CountDownLatch` 实例。
- **异常处理**：如果在调用 `await()` 或 `countDown()` 过程中发生异常，应该正确处理这些异常以避免程序意外终止。

通过合理使用 `CountDownLatch`，可以有效地协调多个线程之间的执行顺序，特别是在有明确依赖关系的任务场景下非常有用。

非常好的问题！我们来深入理解一下 Java 中的 `isAlive()` 方法是如何判断一个线程是否“活着”的。

---

## 🧠 一、什么是“活着”的线程？

在 Java 中，`isAlive()` 方法用于判断某个线程是否处于 **存活状态（alive）**，也就是：

> **该线程已经启动，并且尚未终止。**

换句话说，只要线程还没执行完 `run()` 方法中的代码，或者还没被异常中断或正常退出，它就是“活着的”。

---

## 🔍 二、`isAlive()` 的定义

```java
public final native boolean isAlive();
```

- 这是一个 **native 方法**，由 JVM 实现。
- 它会返回当前线程对象的状态信息。

---

## 💡 三、JVM 是如何知道线程是否“活着”的？

Java 的线程是基于操作系统级别的线程实现的（称为 **1:1 线程模型**），也就是说每个 Java 线程对应一个 OS 线程。

JVM 内部维护了线程的状态信息，这些状态包括但不限于：

| 状态 | 描述 |
|------|------|
| `NEW` | 线程已创建但尚未启动 |
| `RUNNABLE` | 线程正在运行或等待资源（如 CPU） |
| `BLOCKED` | 线程阻塞于锁 |
| `WAITING` | 线程无限期等待另一个线程执行特定操作 |
| `TIMED_WAITING` | 线程在指定时间内等待 |
| `TERMINATED` | 线程已终止 |

### 所以，`isAlive()` 的逻辑本质上是：

> **当线程状态不是 `NEW` 或 `TERMINATED` 时，就认为它是 alive（存活）的。**

---

## ✅ 四、举个例子说明 `isAlive()` 的行为

```java
public class ThreadAliveExample {
    public static void main(String[] args) throws InterruptedException {
        Thread t = new Thread(() -> {
            System.out.println("线程开始运行");
            try {
                Thread.sleep(2000); // 模拟耗时任务
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("线程结束");
        });

        System.out.println("线程未启动：" + t.isAlive()); // false

        t.start();

        System.out.println("线程刚启动：" + t.isAlive()); // true

        t.join(); // 等待线程结束

        System.out.println("线程结束后：" + t.isAlive()); // false
    }
}
```

**输出：**
```
线程未启动：false
线程刚启动：true
线程开始运行
线程结束
线程结束后：false
```

---

## ⚠️ 五、注意事项

| 注意点 | 说明 |
|--------|------|
| `isAlive()` 是瞬态的 | 它只是某一时刻的快照，不能保证调用后线程一定还在运行 |
| 不要依赖 `isAlive()` 做精确同步控制 | 多线程环境下，线程可能在你调用 `isAlive()` 后立即死亡 |
| `isAlive()` 不会抛出异常 | 它是一个安全的方法，可以放心调用 |
| 只有调用了 `start()` 后才可能为 `true` | 否则永远是 `false` |

- `isAlive() == true` 的时间范围是从 `start()` 被调用开始，到线程完全终止之前。
- `isAlive() == false` 表示线程尚未启动或已经死亡。

---

## ✅ 七、总结

| 问题 | 回答 |
|------|------|
| `isAlive()` 如何判断线程是否活着？ | 通过 JVM 维护的线程状态，判断线程是否已启动但尚未终止 |
| 返回 `true` 的条件是什么？ | 线程已启动且还未终止（不在 NEW 或 TERMINATED 状态） |
| 返回 `false` 的情况有哪些？ | 线程尚未启动（NEW）或已终止（TERMINATED） |
| 是否可靠？ | 是，但只代表某一时刻的状态，不能用来做精确控制 |

## join的原理

###  isAlive() 定义：

```java
public final native boolean isAlive();
```

这是 Java 中 `Thread` 类的一个 **本地方法（native）**，用于判断某个线程是否处于“存活”状态。

### 返回值说明：

- **`true`**：表示该线程已经启动，并且还没有运行结束（即线程还在执行中或正在终止但尚未完全死亡）。
- **`false`**：表示线程尚未启动，或者已经运行完毕并终止了。

---

## 🧠 二、在 `join(long millis)` 中的作用

来看你提供的这段简化逻辑：

```java
while (isAlive()) {
    wait(delay);
}
```

这段代码的意思是：

> **只要目标线程还在运行（isAlive() == true），当前线程就一直等待（wait）。**

也就是说，`join()` 的作用是让当前线程 **等待目标线程执行完毕** 后再继续执行。

---

## ✅ 三、举个例子理解 `isAlive()` 和 `join()`

```java
Thread t = new Thread(() -> {
    try {
        System.out.println("线程开始");
        Thread.sleep(2000); // 模拟耗时操作
        System.out.println("线程结束");
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
});

t.start();

System.out.println("t.isAlive() after start: " + t.isAlive()); // true

t.join(); // 主线程等待 t 执行完

System.out.println("t.isAlive() after join: " + t.isAlive()); // false
```

**输出：**
```
线程开始
t.isAlive() after start: true
（等待约2秒）
线程结束
t.isAlive() after join: false
```

## ⚠️ 四、注意事项

1. **线程未启动时调用 `isAlive()` 返回 false**：
   ```java
   Thread t = new Thread(...);
   System.out.println(t.isAlive()); // false
   ```

2. **线程结束后调用 `isAlive()` 也返回 false**。

3. **`isAlive()` 是一个瞬态状态检查**：
   - 它只是一个快照状态，不能保证调用后线程一定还在运行。
   - 多线程环境下，线程可能在调用 `isAlive()` 后立即结束。

4. **不是阻塞方法**：
   - `isAlive()` 只是读取线程的状态，不会导致线程挂起。

| `join()` | 当前线程等待目标线程结束，底层通过循环调用 `isAlive()` + `wait()` 实现 |

## 🧠 一、什么是 `park` 和 `unpark`

`park` 和 `unpark` 是 Java 中用于线程阻塞与唤醒的底层机制，由 `java.util.concurrent.locks.LockSupport` 类提供。它们是比传统的 `Object.wait()` / `notify()` 更轻量、更灵活的线程控制方式。

### 方法定义：
```java
public class LockSupport {
    public static void park();             // 阻塞当前线程
    public static void unpark(Thread thread); // 唤醒指定线程
}
```

---

## 🔍 二、核心原理：许可（Permit）机制

`park` 和 `unpark` 的实现基于一个叫做 **“许可”（permit）** 的概念：

| 操作 | 行为 |
|------|------|
| `unpark(Thread t)` | 给线程 `t` 发放一个“许可”（最多只能有一个）。如果线程已经处于等待状态，则它会被唤醒；如果没有等待，则下次调用 `park()` 时会直接返回，不阻塞。 |
| `park()` | 如果当前线程有可用的“许可”，则立即返回，并消耗这个许可；如果没有许可，则线程进入等待状态，直到其他线程调用 `unpark` 或者线程被中断。 |

### 示例说明：

```java
Thread t = new Thread(() -> {
    System.out.println("线程开始");
    LockSupport.park(); // 等待许可
    System.out.println("继续执行");
});

t.start();

// 主线程先给 t 发一个许可
LockSupport.unpark(t);

// 再次调用 park() 会立刻返回（因为之前已经有许可）
LockSupport.park();
```

输出：
```
线程开始
继续执行
```

> 注意：即使 `unpark()` 在 `park()` 之前调用，也能生效！

## 🧩 三、和 wait/notify 的区别

| 特性 | `park/unpark` | `wait/notify` |
|------|----------------|----------------|
| 是否需要持有锁 | ❌ 不需要 | ✅ 必须在 synchronized 块中调用 |
| 允许提前唤醒 | ✅ 可以先调用 `unpark` | ❌ 必须先 `wait` 后 `notify` |
| 多次调用 unpark | ❌ 无效（只保留一个许可） | N/A |
| 中断响应 | ✅ 被中断后会返回 | ✅ 会抛出 InterruptedException |
| 精确唤醒 | ✅ 可以指定具体线程 | ❌ 只能唤醒一个或全部等待线程 |
| 底层实现 | JVM 提供支持（C++ 实现） | 基于对象监视器（monitor）机制 |

## ⚙️ 四、底层实现原理（JVM 层面）

`park` 和 `unpark` 是 JVM 提供的本地方法，其底层依赖操作系统提供的线程调度机制（如 Linux 的 `futex`、Windows 的 `Condition Variable`）。

### 1. 许可（Permit）的存储位置

每个线程内部维护了一个布尔变量（可以理解为一个信号量），表示是否有许可：

- 初始值为 0。
- 调用 `unpark` 将其设为 1。
- 调用 `park` 时，如果为 1，则将其设为 0 并立即返回；否则挂起线程。

### 2. 阻塞与唤醒过程

- 当线程调用 `park()` 时，JVM 会检查它的 permit：
  - 如果为 1，就清空并返回；
  - 如果为 0，就将线程放入一个等待队列中，并挂起。

- 当调用 `unpark(Thread t)` 时：
  - 如果线程 t 正在等待，则唤醒它；
  - 如果线程未等待，则保存一个 permit，供下次 `park()` 使用。

### 3. 不受同步块限制

不同于 `wait()` 必须在 `synchronized` 块中使用，`park()` 可以随时调用，这使得它更适合用于构建高级并发工具。

`park/unpark` 主要用于构建以下类型的并发工具：

| 工具类 | 使用场景 |
|--------|----------|
| `AbstractQueuedSynchronizer (AQS)` | 构建 `ReentrantLock`, `CountDownLatch`, `Semaphore` 等 |
| `FutureTask` | 异步任务执行与结果获取 |
| `ForkJoinPool` | 并行任务调度与工作窃取 |
| 自定义线程池/协程库 | 控制线程生命周期与协作 |

## 🧪 六、示例代码：模拟 Condition.await() / signal()

下面是一个使用 `park/unpark` 实现的简单生产者消费者模型：

```java
import java.util.concurrent.locks.LockSupport;

public class ParkUnparkExample {
    private static volatile boolean hasData = false;
    private static Thread consumerThread;

    public static void main(String[] args) throws InterruptedException {
        consumerThread = new Thread(() -> {
            while (!hasData) {
                System.out.println("消费者等待数据...");
                LockSupport.park(); // 等待通知
            }
            System.out.println("消费数据");
        });

        consumerThread.start();

        Thread.sleep(1000); // 模拟延迟

        System.out.println("生产者发送数据");
        hasData = true;
        LockSupport.unpark(consumerThread); // 唤醒消费者
    }
}
```
