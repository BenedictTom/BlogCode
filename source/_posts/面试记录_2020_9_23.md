## 一、Java 核心与高并发（10分钟）

### 1.1 `ThreadPoolExecutor` 的核心参数？如果任务过多，队列满了会怎样？

**参考答案：**

- **核心参数**：
  - `corePoolSize`：核心线程数，常驻内存
  - `maximumPoolSize`：最大线程数（临时线程）
  - `keepAliveTime` + `unit`：非核心线程空闲存活时间
  - `workQueue`：任务队列（如 `LinkedBlockingQueue`）
  - `threadFactory`：线程创建工厂
  - `handler`：拒绝策略

- **队列满后的处理流程**：
  1. 当前线程数 < `maximumPoolSize` → 创建新线程执行任务
  2. 已达 `maximumPoolSize` 且队列满 → 触发**拒绝策略**：
     - `AbortPolicy`（默认）：抛出 `RejectedExecutionException`
     - `CallerRunsPolicy`：由提交任务的线程直接执行（降级）
     - `DiscardPolicy`：静默丢弃
     - `DiscardOldestPolicy`：丢弃队列最老任务，重试提交

> ✅ 加分点：提到“线上推荐使用 `CallerRunsPolicy` 防止雪崩”

### 1.2 简述 JVM 内存结构？各区域作用？

**参考答案：**

JVM 内存分为：

| 区域 | 作用 | 是否线程私有 |
|------|------|-------------|
| **程序计数器** | 记录当前线程执行字节码的位置 | ✅ 是 |
| **虚拟机栈** | 存放方法调用栈帧（局部变量、操作数栈） | ✅ 是 |
| **本地方法栈** | Native 方法调用 | ✅ 是 |
| **堆（Heap）** | 对象实例、数组存储区域 | ❌ 否 |
| **方法区（元空间）** | 类信息、常量、静态变量、JIT 编译代码 | ❌ 否 |

> ✅ 注意：JDK 8 后“永久代”被“元空间（Metaspace）”取代，使用本地内存。

### 1.3 `ThreadLocal` 内存泄露的原因？如何避免？

**参考答案：**

- **原因**：
  - `ThreadLocal` 的 `Entry` 是 `WeakReference<K>`，但 `value` 是强引用
  - 线程长期运行（如线程池），`ThreadLocal` 被回收后，`value` 仍被 `Thread` 的 `threadLocals` map 引用 → 无法回收 → 内存泄露

- **解决方案**：
  1. 每次使用后手动调用 `remove()`：
     ```java
     try {
         threadLocal.set(value);
         // ...
     } finally {
         threadLocal.remove(); // 必须！
     }
     ```
  2. 使用 `try-finally` 或 `AutoCloseable` 封装
  3. 避免 `static ThreadLocal` 长期持有

> ✅ 加分点：提到“Tomcat 中因未 remove 导致 PermGen 溢出”的真实案例

### 1.4 `ConcurrentHashMap` 如何实现线程安全？JDK 7 和 8 的区别？

**参考答案：**

- **JDK 7**：
  - 分段锁（`Segment`），每个 Segment 相当于一个 `ReentrantLock + HashEntry[]`
  - 并发度 = Segment 数（默认 16）
  - 锁粒度大，性能一般

- **JDK 8**：
  - 放弃分段锁，使用 `synchronized + CAS + volatile`
  - Node 数组 + 链表/红黑树
  - 锁粒度降到**单个桶（bin）**
  - 插入时用 CAS 尝试，失败再 `synchronized`

> ✅ 优势：JDK 8 更高效，尤其在低冲突场景

### 1.5 `synchronized` 和 `ReentrantLock` 的区别？

| 对比项 | `synchronized` | `ReentrantLock` |
|--------|----------------|-----------------|
| 实现 | JVM 层面（Monitor） | JDK 层面（AQS） |
| 灵活性 | 低（自动加锁/释放） | 高（可中断、超时、公平锁） |
| 公平性 | 非公平 | 可设置公平锁 |
| 条件变量 | `wait/notify` | `Condition`（更灵活） |
| 锁获取信息 | 无法判断是否持有锁 | 可通过 `isHeldByCurrentThread()` 判断 |

> ✅ 使用建议：
> - 简单场景 → `synchronized`
> - 复杂控制（如超时、中断）→ `ReentrantLock`

### 1.6 AOP 是如何实现的？JDK 动态代理和 CGLIB 有何区别？

**参考答案：**

  - Spring AOP 基于**动态代理**

## 二、数据库与缓存（8分钟）

### 2.1 MySQL 事务的特性（ACID）？在 Spring 中事务失效的几种情况？

**参考答案：**

- **ACID**：
  - **A**tomicity：原子性（全成功或全失败）
  - **C**onsistency：一致性（状态合法）
  - **I**solation：隔离性（并发控制）
  - **D**urability：持久性（写入磁盘）

- **Spring 事务失效场景**：
  1. **方法非 public**：`@Transactional` 只对 public 方法有效
  2. **自调用**：同类中 `this.method()` 调用，绕过代理
  3. **异常被捕获未抛出**：`try-catch` 吃掉异常
  4. **异常类型不匹配**：默认只回滚 `RuntimeException`
  5. **数据源未配置事务管理器**
  6. **Propagation 配置错误**：如 `NOT_SUPPORTED`

> ✅ 解决：使用 `AopContext.currentProxy()` 或重构调用逻辑

### 2.2 MySQL 索引的设计思路？什么是覆盖索引？如何优化慢查询？

**参考答案：**

- **索引设计原则**：
  - 高频查询字段建索引
  - 联合索引遵循最左前缀
  - 避免过多索引（影响写性能）
  - 选择性高的字段优先（如 status 不适合单独建索引）

- **覆盖索引（Covering Index）**：
  - 查询字段全部包含在索引中，无需回表
  - 例如：`idx_name_age(name, age)`，查询 `SELECT name, age FROM t WHERE name=?` 可覆盖

- **慢查询优化步骤**：
  1. 开启 `slow_query_log`
  2. 用 `EXPLAIN` 分析执行计划
  3. 检查是否走索引、有无全表扫描、`type=ALL`
  4. 优化 SQL 或添加索引
  5. 必要时分库分表

### 2.4 如何实现分布式锁？Redis 和 ZooKeeper 方案对比？

**参考答案：**

- **Redis 实现**：
  - `SET key value NX PX 30000`（原子设置）
  - 使用 `Redisson` 的 `RLock`（支持可重入、看门狗自动续期）
  - 缺点：主从切换可能丢锁（CP 不强）

- **ZooKeeper 实现**：
  - 创建临时顺序节点，最小节点获得锁
  - Watch 机制监听前一个节点
  - 强一致性（CP），但性能较低

| 对比 | Redis | ZooKeeper |
|------|-------|-----------|
| 性能 | 高 | 低 |
| 一致性 | AP（最终一致） | CP（强一致） |
| 复杂度 | 低 | 高 |
| 适用场景 | 高并发、容忍偶尔失效 | 金融级强一致 |

> ✅ 推荐：普通业务用 Redisson，金融用 ZooKeeper

### 3.1 RocketMQ 事务消息如何实现？如何保证消息不丢失？

**参考答案：**

- **事务消息流程**：
  1. 发送“半消息”（Prepared）
  2. 执行本地事务
  3. 根据结果提交或回滚（Commit/Rollback）

- **保证不丢失**：
  - **生产者**：同步发送 + 重试机制
  - **Broker**：同步刷盘 + 主从复制（SYNC_MASTER）
  - **消费者**：手动 ACK，消费成功再确认

> ✅ 项目实践（5G消息平台）：
> - 用事务消息保证“积分发放”与“消息发送”一致性
> - 消费失败写入死信队列 + 人工干预

```java
import java.util.*;
import java.util.concurrent.*;
import java.util.concurrent.locks.ReentrantLock;

/**
 * 线程安全的本地缓存，支持TTL和LRU淘汰
 */
public class LRUCache<K, V> {
    private final int capacity;
    private final Map<K, CacheEntry<V>> map;
    private final LinkedHashMap<K, Long> lruList;
    private final ReentrantLock lock = new ReentrantLock();
    private final ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);
    private volatile boolean isRunning = true;

    static class CacheEntry<V> {
        final V value;
        final long expireTimeMs;

        CacheEntry(V value, long ttlMs) {
            this.value = value;
            this.expireTimeMs = System.currentTimeMillis() + ttlMs; // ✅ 正确计算过期时间
        }

        boolean isExpired() {
            return System.currentTimeMillis() > expireTimeMs; // ✅ 判断是否已过期
        }
    }

    public LRUCache(int capacity) {
        if (capacity <= 0) {
            throw new IllegalArgumentException("Capacity must be positive");
        }
        this.capacity = capacity;
        this.map = new HashMap<>();
        // ✅ 初始化 LinkedHashMap，启用 accessOrder=true 实现 LRU
        // 重写 removeEldestEntry，在 size > capacity 时触发淘汰
        this.lruList = new LinkedHashMap<K, Long>(16, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry<K, Long> eldest) {
                if (map.size() > capacity) {
                    // 淘汰最久未使用的 key
                    map.remove(eldest.getKey());
                    return true;
                }
                return false;
            }
        };
    }

    /**
     * 获取缓存值，若不存在或已过期则返回null
     * 注意：get操作应更新访问顺序（触发LRU机制）
     */
    public V get(K key) {
        lock.lock();
        try {
            CacheEntry<V> entry = map.get(key);
            if (entry == null) {
                return null;
            }
            if (entry.isExpired()) {
                // ✅ 过期则清理并返回 null
                map.remove(key);
                lruList.remove(key);
                return null;
            }
            // ✅ 更新访问顺序（put 触发 accessOrder）
            lruList.put(key, System.currentTimeMillis());
            return entry.value;
        } finally {
            lock.unlock();
        }
    }

    /**
     * 写入缓存，带TTL（毫秒）
     * 若已存在则更新；若超出容量且新key不存在，则淘汰最久未使用的条目
     */
    public void put(K key, V value, long ttlMs) {
        if (ttlMs <= 0) {
            throw new IllegalArgumentException("TTL must be positive");
        }
        CacheEntry<V> newEntry = new CacheEntry<>(value, ttlMs);
        lock.lock();
        try {
            map.put(key, newEntry);
            // ✅ put 到 lruList 会触发 accessOrder 和 removeEldestEntry
            lruList.put(key, System.currentTimeMillis());
        } finally {
            lock.unlock();
        }
    }

    /**
     * 返回当前缓存中未过期的条目数量
     */
    public int size() {
        lock.lock();
        try {
            // 注意：这里不主动清理过期项，仅返回当前 map 大小
            return map.size();
        } finally {
            lock.unlock();
        }
    }

    /**
     * 手动清理所有过期条目
     * @return 被清理的条目数量
     */
    public int cleanupExpiredEntries() {
        lock.lock();
        try {
            int count = 0;
            Iterator<Map.Entry<K, CacheEntry<V>>> it = map.entrySet().iterator();
            while (it.hasNext()) {
                Map.Entry<K, CacheEntry<V>> entry = it.next();
                if (entry.getValue().isExpired()) {
                    it.remove(); // ✅ 从 map 中移除
                    lruList.remove(entry.getKey()); // ✅ 同步从 lruList 中移除
                    count++;
                }
            }
            return count;
        } finally {
            lock.unlock();
        }
    }

    /**
     * 启动后台任务：每30秒自动清理过期条目
     */
    public void startCleanupScheduler() {
        scheduler.scheduleAtFixedRate(() -> {
            if (!isRunning) return;
            cleanupExpiredEntries(); // ✅ 调用清理方法
        }, 30, 30, TimeUnit.SECONDS);
    }

    /**
     * 优雅关闭资源
     */
    public void shutdown() {
        isRunning = false;
        scheduler.shutdown();
        try {
            if (!scheduler.awaitTermination(5, TimeUnit.SECONDS)) {
                scheduler.shutdownNow();
            }
        } catch (InterruptedException e) {
            scheduler.shutdownNow();
            Thread.currentThread().interrupt();
        }
    }

    @Override
    public String toString() {
        lock.lock();
        try {
            return String.format("LRUCache{size=%d, capacity=%d}", map.size(), capacity);
        } finally {
            lock.unlock();
        }
    }

    /**
     * 测试用例
     */
    public static void main(String[] args) throws InterruptedException {
        LRUCache<String, String> cache = new LRUCache<>(3);
        cache.startCleanupScheduler();

        cache.put("key1", "value1", 5000);  // 5秒过期
        cache.put("key2", "value2", 3000);  // 3秒过期
        cache.put("key3", "value3", 10000); // 10秒过期

        System.out.println("Initial size: " + cache.size()); // 输出 3

        Thread.sleep(4000);
        System.out.println("After 4s, key2 should be gone: " + cache.get("key2")); // null
        System.out.println("Current size: " + cache.size()); // 可能为 2（key2 已过期）

        cache.put("key4", "value4", 5000); // 可能触发 LRU 淘汰 key1（最久未使用）
        System.out.println("After put key4, size: " + cache.size()); // 应为 3

        cache.shutdown();
    }
}
```
